### Результаты разведки API (03.07, факты)

**Yandex AI Studio** (ключ проверен вживую):
- Доступны: `yandexgpt-lite/latest` (в. 25.03.2025), `yandexgpt/latest` (Pro), `yandexgpt-32k/latest`; llama-моделей нет.
- **Квота: 10 одновременных сессий на ключ (gauge), общая** — при 10 параллельных потоках ловим 429 (~20% запросов). Стратегия: конкурентность 5–8 + экспоненциальный backoff-ретрай; 429 — штатная ситуация, не ошибка.
- **Async-эндпоинт `completionAsync` работает** (операции ~3 с) — отдельные квоты, дешевле; основной путь для массовой экстракции чанков.
- **Structured output `jsonSchema` работает** (проверено) — экстракция без парсинга свободного текста.
- Эмбеддинги `text-search-doc/query`: 256-dim, работают. Латентность lite ~1.2 с.

**Корпус / OCR** (выборка 200 из 1161 PDF):
- Полных сканов **0%** — почти весь корпус с текстовым слоем; ~8% PDF имеют 10–50% «пустых» страниц (обложки, рекламные полосы, диаграммы).
- OCR — минорный путь: только страницы без текстового слоя, а не целые документы. Бэклог OCR мал, GPU-сервис справится за часы.
- **docling**: 0.6–1.2 с/стр, корректный Markdown (`{success, markdown, error, filename}`). **marker** — обрыв на 30 с (прокси-таймаут), пустой ответ — исключён.

Принципы качества (зафиксированы):
1. **Assertion-центричная модель** — утверждения с evidence, confidence, review_status; не «голые рёбра».
2. **Rule-first для чисел** — число, отсутствующее дословно в источнике, в граф не попадает.
3. **Терминология как продукт** — реестр концептов RU/EN, всё через concept_id.
4. **Filter-first retrieval + RRF + graph expansion**; LLM синтезирует только из retrieved-доказательств.
5. **Документы — не query-time субстрат**: всё дорогое предвычисляется.

## 2. Онтология (единая, фиксируется в ontology.ttl)

### 2.1 Типы узлов

| Тип | Примеры | Ключевые свойства |
|---|---|---|
| `Material` | никель, катодная медь, техногенный гипс, штейн, шлак, католит | concept_id, name, name_en, aliases[], class (metal/ore/reagent/waste/solution) |
| `Process` | электроэкстракция, кучное выщелачивание, флотация, обессоливание | concept_id, name, name_en, aliases[], domain (hydro/pyro/обогащение/экология/горное дело/водоочистка) |
| `Equipment` | ванна электроэкстракции, ПВП, диафрагменная ячейка | concept_id, name, name_en, aliases[] |
| `Parameter` | концентрация сульфатов, температура, скорость циркуляции | concept_id, name, unit_canonical |
| `Condition` | «сульфаты ≤300 мг/л», «холодный климат», «pH 2–4» | param, op (=,<,>,<=,>=,range), value, value2, unit, qualitative |
| `Measurement` | «извлечение 92,5 %», «расход 1,2 м³/ч» | param, value, unit, uncertainty, context |
| `Experiment` | опыт обеднения шлака при T=1250°C | title, date, facility, status |
| `Publication` | статья, обзор, доклад, отчёт, патент | doc_id, title, year, source_type, section, journal, lang, geography (RU/foreign/global) |
| `Expert` | Косов Я.И. | name, affiliation, topics[] |
| `Facility` | ЛГМ, ЛПМ, ИАЦ, Институт Гипроникель, з-д Уэльва | name, type (lab/plant/institute), country |
| `Assertion` | «оптимальная скорость циркуляции католита X м³/ч» | statement, confidence (high/medium/low), n_sources, review_status (auto/confirmed/disputed/rejected), evidence: [{doc_id, chunk_id, quote}], version, valid_from, superseded_by |

> Факты = Assertion-объекты с доказательствами, условиями и провенансом. Условия и измерения — только структурированные, никогда «просто текст». Версионирование: новый вывод не затирает старый, а создаёт версию с `superseded_by`.

### 2.2 Типы связей

`uses_material` (Process→Material) · `produces_output` (Process→Material) ·
`operates_at_condition` (Process/Experiment→Condition) · `uses_equipment` (Process→Equipment) ·
`measured` (Experiment→Measurement) · `described_in` (любой→Publication) ·
`authored_by` (Publication→Expert) · `works_at` (Expert→Facility) ·
`expert_in` (Expert→Process/Material) · `validated_by` (Assertion→Experiment/Publication) ·
`contradicts` (Assertion↔Assertion) · `supersedes` (Assertion→Assertion, версии) ·
`located_in` (Facility→страна/регион) · `about` (Publication→Material/Process)

Каждое ребро: `source_doc`, `chunk_id`, `confidence` (0–1), `extracted_at`, `method` (rule/llm/manual), `created_by` (pipeline/эксперт — для ручных правок).

## 5. Направления работ и чеклисты агентов

### 🅰 Направление A — Ingest (`pipeline/ingest/`) — 2 агента

**A-core (пайплайн):**
- [ ] A1. Сканер дерева → `corpus/manifest.jsonl`; распаковка zip/rar/7z + многотомников (.001/.002), кириллические кодировки имён, защита от бомб
- [ ] A2. Метаданные из путей: раздел/журнал/год/source_type/geography_hint (набор эвристик + таблица исключений)
- [ ] A3. Текст: PyMuPDF (pdf), python-docx + mammoth + LibreOffice-fallback (doc/docm), python-pptx, openpyxl/xlrd (xls: листы → текстовые таблицы)
- [ ] A4. Чанкование ~1000 токенов, overlap 100, по границам абзацев/заголовков; язык (pycld3); фильтр мусора (колонтитулы, оглавления, реклама)
- [ ] A5. Идемпотентный CLI `python -m ingest --wave N`; инкрементальность (хэш файла); отчёт ok/failed/ocr_queued
- [ ] A6. Прогон волн 1→4 по графику §3

**A-ocr (сканы и сложные форматы):**
- [ ] A7. Детектор постранично: страницы без текстового слоя → OCR только этих страниц (полных сканов в корпусе ~0%, частично «пустых» PDF ~8%)
- [ ] A8. Клиент pdf-ocr-api: **docling** (0.6–1.2 с/стр, проверено; marker исключён — таймаут), параллелизм 2–3, ретраи, дисковый кэш; большие PDF — посекционно
- [ ] A9. Локальный fallback tesseract (rus+eng) на случай недоступности сервиса
- [ ] A10. Прогон OCR всего бэклога сканов; выборочная проверка качества (10 документов)
- [ ] A11. `corpus/README.md`: статистика, покрытие, проблемные файлы

DoD: ≥97% файлов волн 1–2 и ≥90% волн 3–4 в статусе ok; сканы OCR-нуты; повторный запуск ничего не дублирует.

### 🅱 Направление B — Извлечение знаний (`pipeline/extract/`, `shared/`) — 3 агента

**B-rules (детерминированный слой):**
- [ ] B1. Библиотека паттернов: числа+единицы+компараторы+диапазоны («≤300 мг/л», «200–300 мг/л», «от 100 т/сут», «1250±20 °C»), даты, проценты извлечения, ТЭП
- [ ] B2. Канонизация единиц (pint + кастомная таблица: мг/л=мг/дм³, г/т, % изв., м³/ч…)
