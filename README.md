# Научный клубок

**Карта знаний R&D для горно-металлургической отрасли.** Задаёте вопрос на естественном
языке (рус/англ) — получаете структурированный ответ **со ссылками на первоисточники,
проверяемыми цифрами, подсветкой противоречий и «пустых зон»**.

**Живое демо: https://158-255-4-151.sslip.io** (HTTPS через Caddy; запасной адрес — http://158.255.4.151)

## Проблема

Компания за 90+ лет накопила отчёты, регламенты, статьи, патенты на четырёх языках. Сегодня
найти нужное — «ручной квест по архивам»: теряется память компании, дублируются эксперименты,
учёный тратит на **поиск** больше времени, чем на анализ. Нужна не БД, а живая карта знаний,
которая связывает материалы/процессы/установки/выводы, оперирует цифрами (концентрации,
температуры), сравнивает отечественную и зарубежную практику и **не прячет** противоречия и
пробелы. Каждый факт должен быть проверяемым — со ссылкой на источник.

## Что делает «Научный клубок»

- **NL-поиск с evidence packet**: markdown-ответ, синтезированный ТОЛЬКО из найденных
  доказательств, с цитатами `[n]` → карточка первоисточника с дословным фрагментом.
- **Проверяемые числа**: значения (концентрации, температуры, расходы) извлечены rule-first и
  подсвечены; число, отсутствующее дословно в источнике, в ответ не попадает.
- **Гейт честности**: по теме вне корпуса система отвечает «доказательств не найдено», а не
  галлюцинирует (honesty на adversarial — 100 % на референс-прогоне).
- **Панель «Как получен ответ»**: объяснимость — какие ветки поиска сработали, какие
  документы прошли гейт, из чего синтезирован ответ.
- **Граф знаний**: интерактивный подграф ответа, противоречия (`contradicts`), «пустые зоны».
- **Масштабирование на новые данные**: загрузка документа через UI (upload работает
  end-to-end) → извлечение фактов → фрагмент графа → слияние с общим графом; корпус
  обрабатывается волнами.
- **Независимость от облака**: LLM-шлюз (Yandex / Groq / GigaChat / OpenRouter / OpenAI /
  локальные) и эмбеддинг-«пространства» (облако или собственный GPU) меняются конфигом.

## Быстрый старт (30 секунд, готовые контейнеры)

Жюри «щупает» стенд без сборки проекта — поднимается вся система одной командой:

```bash
cp .env.example .env          # задать NEO4J_PASSWORD (LLM-ключ опционален — есть фоллбэки)
docker compose -f deploy/docker-compose.prod.yml up -d --build
docker compose -f deploy/docker-compose.prod.yml ps   # neo4j + es + api + ui + nginx
```

Откройте `http://localhost` (или публичный IP стенда). Всё за nginx: `/` → UI, `/api` → FastAPI.
Health: `GET /api/health`. Подробности деплоя — `deploy/README.md`.

Локальная разработка (dev-хранилища + uvicorn) — см. `backend/README.md`.

## Архитектура

```
   ИСХОДНЫЕ ДАННЫЕ (~1.5 тыс. файлов, ~4.9 ГБ: PDF/DOCX/ZIP/XLS/PPTX, RU/EN/…)
              │
   [A] INGEST │  обход+распаковка → извлечение текста (PyMuPDF/docx/pptx/xlsx)
              │  → OCR постранично (docling) → чанкинг ~1000 ток. (жёсткий кап)
              ▼  corpus/{documents,chunks}.jsonl
   [B] EXTRACTION  rule-first числа/единицы/гео  +  LLM: сущности, связи,
              │    Assertion c дословной цитатой  →  реестр концептов RU/EN
              ▼  graph/{nodes,edges}.jsonl + embeddings
   [C] GRAPH + SEARCH
              │   Neo4j 5 (граф, обходы, provenance)  ‖  Elasticsearch 8 (full-text, range)
              │   query-planner (NL→интент) → filter-first retrieval → RRF-слияние
              │   → graph expansion → гейт честности → evidence-packet синтез
              ▼  FastAPI REST
   [D] UI     React + TS + Cytoscape: NL-поиск, evidence packet, цитаты,
              │  граф, фильтры, дашборд, сравнение, загрузка документов
              ▼
          nginx (:80) → браузер / скринкаст-демо
```

Стек: **Neo4j 5** (+ vector index) · **Elasticsearch 8** (RU+EN морфология, range) ·
**FastAPI** (Python 3.12) · **React + Vite + TS + Tailwind + Cytoscape** · **docker compose +
nginx + Caddy (HTTPS)**. Данные в цифрах: корпус **1320 документов / 116 801 чанк** (волна 4
дописывается), граф **28 159 узлов / 47 722 ребра** (SHACL-валидация — `docs/VALIDATION_REPORT.md`).

**Мультипровайдерность.** LLM — через шлюз `shared/llm_gateway.py`: Yandex / Groq / GigaChat
(OAuth) / OpenRouter / OpenAI / локальные (vLLM, Ollama) — фолбэк-цепочки, пер-роль модели,
смена — правкой `.env`. Эмбеддинги — через `shared/embeddings_gateway.py` («пространства»):
полный корпус (113 589 чанков × 1024-dim) пересчитан **локально на GPU** заказчика
(`Qwen3-Embedding-0.6B`, 32K-контекст — выбран вместо e5 с лимитом 512 токенов). Решение не
привязано ни к одному облаку и ресурсо-оптимально.

## Семантический слой (FAIR)

Формальная семантика на стандартах W3C — граф портируем и машинно-валидируем.

- **OWL-онтология** — [`docs/ontology.ttl`](docs/ontology.ttl): 12 классов (Material, Process, Condition, Measurement, Assertion, Publication…), 17 object- и 29 datatype-свойств; метки RU+EN, комментарии RU, домены/диапазоны.
- **SHACL-шейпы** — [`docs/shapes.ttl`](docs/shapes.ttl): обязательный провенанс факт-рёбер (source_doc, confidence 0..1, method∈{rule,llm,manual}), структура Condition/Measurement (op, value:double, unit), Assertion (statement, confidence, review_status, непустой evidence).
- **Реальная валидация** — `python backend/validate_graph.py --full` (rdflib → pyshacl): весь граф **28 159 узлов / 47 722 ребра → 722 300 триплетов** за ~14 с. Провенанс рёбер и Assertion проходят чисто; фактический отчёт с топ-категориями нарушений — [`docs/VALIDATION_REPORT.md`](docs/VALIDATION_REPORT.md).
- **Единый словарь** — JSON-LD экспорт (`backend/app/exporters.py`, schema.org + PROV-O) использует тот же базовый IRI `https://sci-tangle.nornickel.example/ontology#`, что и TTL: онтология, валидация и экспорт — один словарь.

## Дополнительно (бонус сверх ядра)

Экспорт (md / JSON-LD+PROV-O / PDF / xlsx), подписки на темы, ручная правка графа
(human-in-the-loop), доменные сводки, аудит, сравнительные таблицы технологий.
**Разграничение доступа (RBAC/ABAC, 5 ролей)** реализовано как бонус, но по уточнению
заказчика на Q&A эта задача **отменена** — оставлено как демонстрационная доп. возможность,
не ядро продукта.
