# Научный клубок

**Карта знаний R&D для горно-металлургической отрасли.** Задаёте вопрос на естественном
языке (рус/англ) — получаете структурированный ответ **со ссылками на первоисточники,
проверяемыми цифрами, подсветкой противоречий и «пустых зон»**.

> Хакатон Норникеля / Гипроникеля, задача «Единая карта знаний R&D». Кодовое имя: `sci-tangle`.

**Живое демо: http://158.255.4.151**

---

## Проблема

Компания за 90+ лет накопила отчёты, регламенты, статьи, патенты на четырёх языках. Сегодня
найти нужное — «ручной квест по архивам»: теряется память компании, дублируются эксперименты,
учёный тратит на **поиск** больше времени, чем на анализ. Нужна не БД, а живая карта знаний,
которая связывает материалы/процессы/установки/выводы, оперирует цифрами (концентрации,
температуры), сравнивает отечественную и зарубежную практику и **не прячет** противоречия и
пробелы. Каждый факт должен быть проверяемым — со ссылкой на источник.

## Что делает «Научный клубок»

- **NL-поиск с evidence packet**: markdown-ответ, синтезированный ТОЛЬКО из найденных
  доказательств, с цитатами `[n]` → карточка первоисточника с дословным фрагментом.
- **Проверяемые числа**: значения (концентрации, температуры, расходы) извлечены rule-first и
  подсвечены; число, отсутствующее дословно в источнике, в ответ не попадает.
- **Гейт честности**: по теме вне корпуса система отвечает «доказательств не найдено», а не
  галлюцинирует (honesty на adversarial — 100 % на референс-прогоне).
- **Граф знаний**: интерактивный подграф ответа, противоречия (`contradicts`), «пустые зоны».
- **Масштабирование на новые данные**: загрузка документа → извлечение фактов → фрагмент
  графа → слияние с общим графом; корпус обрабатывается волнами.

## Быстрый старт (30 секунд, готовые контейнеры)

Жюри «щупает» стенд без сборки проекта — поднимается вся система одной командой:

```bash
cp .env.example .env          # задать NEO4J_PASSWORD (LLM-ключ опционален — есть фоллбэки)
docker compose -f deploy/docker-compose.prod.yml up -d --build
docker compose -f deploy/docker-compose.prod.yml ps   # neo4j + es + api + ui + nginx
```

Откройте `http://localhost` (или публичный IP стенда). Всё за nginx: `/` → UI, `/api` → FastAPI.
Health: `GET /api/health`. Подробности деплоя — `deploy/README.md`.

Локальная разработка (dev-хранилища + uvicorn) — см. `backend/README.md`.

## Архитектура

```
   ИСХОДНЫЕ ДАННЫЕ (1453 файла, ~4.9 ГБ: PDF/DOCX/ZIP/XLS/PPTX, RU/EN/…)
              │
   [A] INGEST │  обход+распаковка → извлечение текста (PyMuPDF/docx/pptx/xlsx)
              │  → OCR постранично (docling) → чанкинг ~1000 ток. (жёсткий кап)
              ▼  corpus/{documents,chunks}.jsonl
   [B] EXTRACTION  rule-first числа/единицы/гео  +  LLM: сущности, связи,
              │    Assertion c дословной цитатой  →  реестр концептов RU/EN
              ▼  graph/{nodes,edges}.jsonl + embeddings
   [C] GRAPH + SEARCH
              │   Neo4j 5 (граф, обходы, provenance)  ‖  Elasticsearch 8 (full-text, range)
              │   query-planner (NL→интент) → filter-first retrieval → RRF-слияние
              │   → graph expansion → гейт честности → evidence-packet синтез
              ▼  FastAPI REST
   [D] UI     React + TS + Cytoscape: NL-поиск, evidence packet, цитаты,
              │  граф, фильтры, дашборд, сравнение, загрузка документов
              ▼
          nginx (:80) → браузер / скринкаст-демо
```

Стек: **Neo4j 5** (+ vector index) · **Elasticsearch 8** (RU+EN морфология, range) ·
**FastAPI** (Python 3.12) · **React + Vite + TS + Tailwind + Cytoscape** · **docker compose + nginx**.
LLM — через мультипровайдерный шлюз (`shared/llm_gateway.py`): Yandex / OpenRouter / OpenAI /
локальные — сменой `.env`. Эмбеддинги — GPU-сервис `Qwen3-Embedding-0.6B` (32K-контекст).
